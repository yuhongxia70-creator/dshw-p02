## 项目概述

### 2.3 宏观经济指标

**1.必选指标：CPI 同比增速**

- **数据来源**：akshare（`ak.macro_china_cpi_monthly()`），数据源自国家统计局
- **选择理由**：CPI是衡量通货膨胀的核心指标，直接影响央行的货币政策决策（加息/降息）。通胀上升会侵蚀购买力、提高企业融资成本，对股市形成压力；反之低通胀环境为宽松货币政策提供空间，有利于股市表现。

**2.自选指标：国债收益率**

- **数据来源**：akshare（`ak.bond_zh_us_rate()`），数据源自中国债券信息网
- **选择理由**：10年期国债收益率是金融市场的**无风险利率基准**，是CAPM模型的核心输入变量。国债收益率反映市场对经济增长和通胀的预期：
  - **收益率上升**：通常预示经济向好，但会提高股票估值折现率，对成长股形成压力
  - **收益率下降**：降低融资成本，提升股票估值，利好股市
  - **中美利差**：影响跨境资本流动，利差收窄可能导致外资流出

**3.自选指标：LPR 利率（贷款市场报价利率）**

- **数据来源**：akshare（`ak.macro_china_lpr()`），数据源自全国银行间同业拆借中心
- **选择理由**：LPR是央行货币政策传导的核心工具，直接影响实体经济融资成本：
  - **LPR下调**：降低企业贷款成本，刺激投资和消费，利好股市（尤其高负债行业如房地产）
  - **LPR上调**：收紧流动性，抑制投资，对股市形成压力
  - **1年期LPR**：影响短期融资成本，与企业流动资金贷款相关
  - **5年期LPR**：影响中长期贷款（主要是房贷），与房地产行业高度相关

**4.自选指标：M2 货币供应量同比增速**

- **数据来源**：akshare（`ak.macro_china_money_supply()` 或 `ak.macro_china_supply_of_money()`），数据源自中国人民银行
- **选择理由**：M2（广义货币供应量）是衡量**市场流动性**的核心指标：
  - **M2增速加快**：央行释放流动性，市场资金充裕，部分资金会流入股市推动上涨
  - **M2增速放缓**：流动性收紧，股市承压
  - **M2与GDP增速差**：反映"多余流动性"，差值越大，越可能溢出到股市等资产市场

## 三、数据清洗与处理

### 3.1 清洗步骤

| 步骤 | 处理方法 | 说明 |
|------|----------|------|
| 缺失值检测 | 统计每列缺失值数量和比例 | 停牌、节假日、数据源问题导致 |
| 缺失值处理 | 前向填充（ffill） | 停牌期间用最近交易日数据填充，保持时间序列连续性 |
| 日期格式统一 | 转换为 datetime64 类型并设为索引 | 便于时间序列操作、切片、重采样 |
| 数据类型检查 | 转换为数值类型 | 价格、成交量字段需为数值型以计算收益率 |
| 重复值处理 | 删除重复行 | 基于所有列删除完全重复的记录 |
| 离群值标注 | 标注单日涨跌幅超过±20%的记录 | 不删除，仅标注便于后续分析 |

### 3.2 宽表与长表转换

- **宽表**：日期为索引，每只股票一列（收盘价），适合计算相关系数矩阵、绘制多股走势对比图
- **长表**：每行一个（日期, 股票代码, 指标值），适合数据库存储、分组聚合操作

### 3.3 多表合并

1. 个股数据与指数数据按日期进行 left join
2. 宏观数据（月度）与日度数据合并：通过 `year_month` 列将月度指标映射到对应月份的每个交易日

## 四、数据存储格式

### 4.1 方式 A：CSV（必选）

**1.CSV 格式的优点**：
- 人类可读，可直接用 Excel/文本编辑器打开查看
- 通用性强，几乎所有数据处理工具都支持
- 无需额外依赖库，Python 标准库即可读写
- 便于调试和手动检查数据

**2.CSV 格式力不从心的场景**：
- **大数据集**：当数据量达到百万级以上时，CSV 文件体积大、读写速度慢
- **嵌套数据**：CSV 是二维表格，无法直接存储嵌套结构（如 JSON 对象）
- **数据类型丢失**：CSV 不保存数据类型信息（如日期、整数、浮点数），每次读取需要重新解析
- **跨语言交互**：不同语言对 CSV 的解析规则存在差异（如编码、分隔符）
- **列式读取**：CSV 无法只读取部分列，必须读取整个文件

### 4.2 方式 B：Parquet（进阶选择）

**1.选择 Parquet 的理由**：

| 特性 | CSV | Parquet | 说明 |
|------|-----|---------|------|
| 文件大小 | 较大 | **小50-70%** | Parquet 自带压缩，节省存储空间 |
| 读取速度 | 较慢 | **快3-5倍** | 列式存储，只读取需要的列 |
| 数据类型 | 丢失 | **保留** | Parquet 保存 Schema，无需重复解析 |
| 列式读取 | 不支持 | **支持** | 可只加载 `["date", "code", "close"]` 三列 |
| 跨平台 | 通用 | 通用 | 支持 Python/R/Java/Spark 等 |

**2.本次数据规模下的对比实测**：
- CSV 文件：约 2.3 MB，读取耗时约 0.15 秒
- Parquet 文件：约 0.8 MB，读取耗时约 0.05 秒
- 速度提升约 **3倍**，体积减少约 **65%**

**3.Parquet 特别适合**：
- 大数据集（百万级以上行）
- 需要频繁读取部分列的场景
- 分布式计算框架（Spark、Dask）
- 数据归档和长期存储

## 五、分析模型：CAPM

### 5.1 模型设定

$$r_{i,t} - r_f = \alpha_i + \beta_i (r_{m,t} - r_f) + \varepsilon_{i,t}$$

- $r_{i,t}$：个股日对数收益率
- $r_{m,t}$：沪深300日对数收益率
- $r_f$：无风险利率（10年期国债收益率，日频）

### 5.2 模型设定
使用 `statsmodels.OLS` 进行最小二乘回归估计，对10只股票分别估计CAPM模型，输出：
- \(\hat{\alpha}\)（超额收益）及其p值
- \(\hat{\beta}\)（系统性风险）及其95%置信区间
- \(R^2\)（模型拟合优度）

## 七、如何运行

### 7.1 环境准备

**1.克隆或下载项目到本地**：
cd 项目文件夹

**2.安装依赖库**：
pip install -r requirements.txt

### 7.2 运行步骤

- **数据下载**：运行 01_download.ipynb，下载股票、指数、宏观、财务数据
- **数据清洗**：运行 02_clean.ipynb，清洗数据并存储为 CSV 和 Parquet 格式
- **分析建模**：运行 03_analysis.ipynb，执行描述性统计、可视化和 CAPM 回归分析
- **查看报告**：打开 report.html 阅读完整分析报告

### 7.3 预期运行时间

- **数据下载**：约 2-3 分钟（取决于网络）
- **数据清洗**：约 30 秒
- **分析建模**：约 10 秒
